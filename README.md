## Подгатовка Программы к работе
### 1. Установка MySQL Workbench
### Скачайте и установите MySQL Workbench с официального сайта: https://dev.mysql.com/downloads/mysql/.

### Убедитесь, что вы устанавливаете версию, совместимую с вашей операционной системой.

### 2. Настройка пользователя и прав доступа на сервере
### Для подключения к MySQL серверу извне необходимо создать пользователя и предоставить ему соответствующие права.
 
### 2.1. Подключение к серверу MySQL
### Подключитесь к серверу MySQL через командную строку или MySQL Workbench с использованием пользователя, у которого есть права администратора (например, root).

### 2.2. Проверка существующих пользователей
### Выполните команду, чтобы проверить, существует ли пользователь и с каких хостов он может подключаться:

### sql
### Copy
### SELECT Host, User FROM mysql.user WHERE User = 'твой_username';
### Замените твой_username на имя пользователя, которое вы планируете использовать для подключения.

### 2.3. Создание пользователя (если его нет)
### Если пользователь с указанным именем и хостом отсутствует, создайте его:

### sql
### Copy
### CREATE USER 'твой_username'@'ваш_host' IDENTIFIED BY 'ваш_пароль';
### Замените:

### твой_username на имя пользователя.

### ваш_host на IP-адрес или имя хоста, с которого будет происходить подключение (например, % для любого хоста или конкретный IP).

### ваш_пароль на пароль для пользователя.

### 2.4. Предоставление прав доступа
### Предоставьте пользователю все необходимые права:

### sql
### Copy
### GRANT ALL PRIVILEGES ON *.* TO 'твой_username'@'ваш_host';
### FLUSH PRIVILEGES;
### *.* означает все базы данных и все таблицы.

### Если нужно ограничить доступ к конкретной базе данных, замените *.* на имя_базы.*.

### 2.5. Проверка прав
### Убедитесь, что права были предоставлены:

### sql
### Copy
### SHOW GRANTS FOR 'твой_username'@'ваш_host';
### 3. Настройка MySQL сервера для внешних подключений
### По умолчанию MySQL сервер может быть настроен на прослушивание только локальных подключений (127.0.0.1). Чтобы разрешить внешние подключения, выполните следующие шаги:

### 3.1. Изменение конфигурации MySQL
### Найдите конфигурационный файл MySQL:

### На Windows: my.ini (обычно находится в C:\ProgramData\MySQL\MySQL Server X.Y\).

### На Linux: /etc/mysql/my.cnf или /etc/my.cnf.

### Откройте файл в текстовом редакторе с правами администратора.
 
### Найдите параметр bind-address и измените его значение:

### ini
### Copy
### bind-address = 0.0.0.0
### Это позволит серверу принимать подключения с любых IP-адресов.

### 3.2. Перезапуск MySQL сервера
### Перезапустите MySQL сервер, чтобы изменения вступили в силу:

### На Windows:

### bash
### Copy
### net stop MySQL
### net start MySQL
### На Linux:

### bash
### Copy
### sudo systemctl restart mysql
### 4. Проверка подключения
### Откройте MySQL Workbench и попробуйте подключиться к серверу, используя:

### Хост: IP-адрес сервера.

### Порт: 3306 (по умолчанию).

### Имя пользователя и пароль, которые вы создали.

### 5. Дополнительные рекомендации
### Безопасность: Не используйте % в качестве хоста, если это не необходимо. Лучше указать конкретный IP-адрес или диапазон.

### Брандмауэр: Убедитесь, что порт 3306 открыт на сервере для входящих подключений.

### Резервное копирование: Перед внесением изменений в конфигурацию или права пользователей сделайте резервную копию данных.

### Если всё сделано правильно, вы сможете подключиться к MySQL серверу с указанного хоста. Если возникнут проблемы, проверьте логи MySQL (error.log) для диагностики.
